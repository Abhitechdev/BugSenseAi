BugSense AI is a full-stack app, not a GitHub Pages-only site.

This URL is now serving a proper landing page instead of rendering the repository README. The interactive BugSense AI product still requires a deployed backend, database, Redis, and vector store to work end to end.

What happened

GitHub Pages serves static files. Because the repo had no index.html, Pages rendered README.md as the site homepage.

What is fixed

The site now has a real homepage, so abhitechdev.github.io/BugSenseAi no longer falls back to the specification document.

How to run the full app

Use the Docker stack locally, or deploy the frontend and backend separately with public services.

Full deployment path

If you want a real public live app, the correct approach is:

  • Frontend: Vercel or Netlify
  • Backend: Render, Railway, or a VPS
  • Database: managed PostgreSQL
  • Redis: managed Redis
  • Vector store: managed service or hosted Chroma alternative
GitHub Pages can host a static homepage for this project, but it cannot run the FastAPI backend or bundled infrastructure behind BugSense AI.
